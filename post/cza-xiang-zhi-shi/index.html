对象的定义和初始化

1.关于using namespace std

using 和namespace是两个关键字

其中std是C++的标准库所驻之命名空间的名称，所谓命名空间是指一种将库名封装起来的方法。其目的是为了避免和应用程序（自己写的新的对象）命名冲突的问题即同名的问题。所以通过using namespace 指定了命名空间就可以解决这样的问题。

std库提供了：语言支持（一些宏及一些标准类型）；输入输出；诊断（异常错误处理）；一般工具；字符串（支持字符串处理的头文件）；容器（定义容器类模板的头文件）；迭代器支持；算法（提供一组基本算法的头文件）；数值操作（提供数值计算的头文件，）；本地化（本地化的头文件）；

2.main实际上不是关键字，只是编译器默认主函数的名字是main

3.初始化的方式

（1）使用 = （assignment运算符进行初始化）沿用c的语法，适用的对象为内置类型(Boolean,integer,floating point,character)或者是对象值可以单一初始化，

string user_name = "Tom";

（2）使用构造函数进行初始化，适用于对象需要多个初始值得情况

eg：complex purei(0,7);

定义并初始化了一个双精度得复数

4.内置得Boolen类型

采用true/false

5.运算符得优先级

NOT：! 优先级最高

逻辑运算符的优先级也很低

assignment：= 优先级最低

注：注意各个数据类型的值域，是否会出现溢出的情况

一些比较重要的数据结构

1.Array 和 Vector

（1）array

array的大小必须是常量表达式，因为数组的空间分配是在链接过后就确定了。

eg: int arr_sample[size_of_samlpe]

（2）vector

首先包含vector的头文件,vector是一个class template

eg: vector vec_sample(size_of_sample)

2.string

string 所占得空间是从堆中动态分配得。与sizof无关，因为sizeof()只是运算符，是编译的时候确定大小的，对动态分配运行过程中的大小是无法计算出正确的结果的。所以在同一库中sizeof任意一个字符串，获得的值是确定的。

1.namespace

namespace(命名空间)：是一种将库名称封装起来的方法。避免发生命名冲突。所谓命名冲突是指，应用程序的两个不同的实体有相同的名称。

使用namespace来解决命名冲突的问题

using namesapce 命名空间名;

这样会使后续的代码将使用指定的命名空间中的名称。

一般使用标准库的命名空间：std
